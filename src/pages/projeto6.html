Visão Geral

Sistema para gerenciamento de quadros de tarefas em Java com persistência de dados via JDBC, MySQL e controle de versão de banco de dados com Liquibase.

Tecnologias Utilizadas

Java
JDBC
MySQL
Liquibase

Descrição Técnica

Aplicação Java voltada à gestão de quadros e tarefas, com persistência realizada via JDBC (driver MySQL) e evolução controlada do esquema de dados com Liquibase. O código separa regras de negócio da camada de acesso a dados para favorecer manutenção, legibilidade e testes.

Principais Funcionalidades

  • Cadastro, atualização e exclusão de quadros e tarefas.
  • Listagem de dados com persistência no MySQL.
  • Controle de versionamento do banco via changelogs do Liquibase.

Arquitetura

Arquitetura monolítica simples com camadas separadas para regras de negócio e persistência (DAO). A base de dados relacional MySQL armazena as informações, enquanto o Liquibase gerencia a evolução do esquema com segurança e rastreabilidade.

Requisitos do Sistema

  • JDK instalado (versão compatível com o projeto).
  • MySQL instalado e acessível.
  • Liquibase configurado (via CLI ou plugin da ferramenta de build).
  • Ferramenta de build Java (Maven ou Gradle) para compilar e executar o projeto.
